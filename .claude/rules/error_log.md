# 错误清单

以下规则从历史错误中提炼,每条都被用户反复纠正过,必须严格遵守。

### 1. 商业分析skill触发(最高优先级)

**触发词**: 用户消息中出现"分析""分析数据""看看数据""复盘"等分析类关键词

**执行策略**:
1. 立刻Read `C:\Users\11825\.claude\skills\data-analysis\skills\business-analyst\SKILL.md`
2. 严格按SKILL.md定义的流程执行,不得跳过任何步骤
3. 先问业务背景,再动手分析

**禁止**: 不得因为system-reminder的available skills列表里没显示就跳过。available skills列表 ≠ 全部skills,自定义skill在`C:\Users\11825\.claude\skills\`目录下,必须主动读取。

### 2. 非平凡代码必须走code-writer + code-reviewer流程

**触发条件**: 写了超过20行的代码,或涉及新模块/重构/核心逻辑修改

**执行策略**:
1. 派code-writer subagent编写代码
2. code-writer完成后,立刻派code-reviewer subagent审查
3. 根据审查结果修复问题
4. 三步连续执行,不等用户催

**禁止**: 写完代码就汇报结果而不跑reviewer。

### 3. 禁止附和式措辞

属于no_ai_style的补充。禁用"说得准""你说对了""确实""没错"等附和用户判断的表达。要么直接接着说内容,要么什么都不说。详见 [rules/no_ai_style.md](rules/no_ai_style.md)。

### 4. 产出文件必须归档到日期文件夹

**触发条件**: 任何分析、报告、脚本等产出文件生成完毕时

**执行策略**:
1. 在工作目录下创建`YYYYMMDD_主题`文件夹(如`20260414_诺特兰德Q1评论分析`)
2. 将本次任务的所有产出(PDF/脚本/图表/数据源)移入该文件夹
3. 工作根目录不允许残留本次产出的散文件

**禁止**: 把PDF、脚本、图表等产出直接堆在工作根目录。

### 5. SKILL.md不得依赖具体代码实现

SKILL.md是行为指引和原则文档,描述做什么和为什么,不引用具体的变量名、函数名、类名等代码实现细节。代码层的API说明放在脚本文件自身的docstring里,SKILL.md只描述原则和流程。

### 6. 所有SKILL.md必须包含no_ai_style规则

**触发条件**: 创建或修改任何SKILL.md时

**执行策略**: 在SKILL.md的禁止/约束章节中,加入no_ai_style的全部规则(自我标榜、解释用意、结尾反问、禁用符号、油腻煽情、表态铺垫、附和措辞、中英文不加空格)。

**禁止**: 创建SKILL.md时遗漏no_ai_style规则。no_ai_style是所有skill的基线约束,不是可选项。

### 7. 代码改完确认后直接打包

**触发条件**: 代码改动完成且用户确认无误后

**执行策略**: 直接执行构建命令(如`cd vue-app && npm run build`),不等用户提醒。

**禁止**: 改完代码后只汇报结果而不打包,等用户来催。

### 8. 禁止AI语感套话

属于no_ai_style的补充。禁用「帮你拆」「帮你梳理」「先记住」「我们来看」「结构化」「算笔账」「铁律」等包装成服务姿态或刻意制造权威感的引导词。直接说内容,不要用这类词做铺垫。

### 9. 错误/反馈规则只写CLAUDE.md,不写memory

**触发条件**: 用户指出错误或给出行为反馈时

**执行策略**: 直接在本文件(CLAUDE.md)对应章节补充或修正规则。

**禁止**: 把行为约定、错误纠正写到memory系统里。memory只存项目相关的状态/偏好,不存行为规则。
